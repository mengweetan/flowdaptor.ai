What's in the file

A real n8n workflow. Not a marketing prompt.

Every purchase delivers a single .json file: the complete n8n workflow, with nodes, connections, and recommended credentials. Import directly into your n8n instance — no FlowDaptor account required, no callback to our servers.

inbox-triage-agent.json 
// Excerpt from a production agent file
{
  "name": "Inbox Triage Agent",
  "version": "1.4.2",
  "nodes": [
    {
      "type": "n8n-nodes-base.gmailTrigger",
      "parameters": { "pollTimes": "everyMinute" }
    },
    {
      "type": "@n8n/n8n-nodes-langchain.openAi",
      "parameters": {
        "model": "gpt-4o-mini",
        "systemPrompt": "Classify: action / fyi / spam / urgent"
      }
    },
    {
      "type": "n8n-nodes-base.switch",
      "parameters": { "rules": [ /* routing logic */ ]  }
    }
  ],
  "credentials_required": ["gmail", "openai", "slack"]
}

FAQ

Common questions before you buy.

Do I need a FlowDaptor account to run the agents?

No. You buy the file, import it into your own n8n instance, and run it. There's no FlowDaptor login, no telemetry, and no callback. The agent talks to the services you configure — never back to us.

Where does my data go?

Wherever you point the agent. Most data stays inside your n8n instance and the services you've connected (e.g. Gmail, your CRM, your LLM provider). FlowDaptor never sees it.

Can I modify the workflows?

Yes. The licence is perpetual and explicitly allows modification. Adapt the nodes, swap models, change routing — it's your workflow once you've bought it.

What's your refund policy?

30 days, no questions, on the Studio Bundle. Individual agents are non-refundable once downloaded (since the file is the product), but we'll fix or replace anything that doesn't work as described.

Do you build custom agents?

Yes — that's a big part of what we do. See the workflow library for catalogue builds, or the enterprise page for embedded engagements.
